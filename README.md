# Бачурин Никита ИТС-2 Лабораторная №8

# Задание 11

## Задача 1

### Текст задачи
Чтение базы данных из бинарного файла

### Алгоритм решения
1. Создаётся класс Photo для хранения данных о фотографии.
2. В классе Photo объявляются закрытые поля: название, автор, год, ширина, высота.
3. Для каждого поля создаются свойства с методами get и set для доступа к данным.
4. Определяется конструктор без параметров для создания пустого объекта.
5. Определяется конструктор с параметрами для инициализации всех полей.
6. Переопределяется метод ToString для формирования строки с данными объекта.
7. Создаётся класс PhotoService для работы с данными.
8. В методе Load создаётся пустой список объектов Photo.
9. Проверяется существование файла по заданному пути.
10. При наличии файла открывается бинарный поток чтения.
11. Выполняется цикл чтения до конца файла.
12. Последовательно считываются значения полей объекта Photo.
13. Создаётся объект Photo и заполняется считанными данными.
14. Объект добавляется в список.
15. После завершения чтения файл закрывается.
16. Возвращается список объектов.

## Задача 2

### Текст задачи
Просмотр базы данных

### Алгоритм решения
1. Используется список объектов Photo, полученный после чтения файла.
2. Выполняется последовательный обход списка.
3. Для каждого элемента вызывается метод ToString.
4. Полученная строка выводится на экран.

## Задача 3

### Текст задачи
Удаление элементов по ключу

### Алгоритм решения
1. Пользователь вводит ключ (название фотографии).
2. Выполняется поиск объекта в списке по совпадению названия.
3. Если объект найден — он удаляется из списка.
4. Если объект не найден — список остаётся без изменений.
5. Обновлённый список сохраняется в файл.

## Задача 4

### Текст задачи
Добавление элементов

### Алгоритм решения
1. Пользователь вводит данные новой фотографии.
2. Ввод проходит через класс InputHelper с проверкой корректности.
3. Создаётся новый объект Photo.
4. Объект добавляется в список.
5. Список сохраняется в бинарный файл.

## Задача 5

### Текст задачи
Реализация запросов

### Алгоритм решения
1. Реализуется метод GetByAuthor.
2. Выполняется выбор объектов с заданным автором.
3. Возвращается список объектов.
4. Реализуется метод GetLargePhotos.
5. Выполняется выбор объектов с шириной больше заданной.
6. Возвращается список объектов.
7. Реализуется метод CountByAuthor.
8. Подсчитывается количество объектов с заданным автором.
9. Возвращается число.
10. Реализуется метод GetLatestPhoto.
11. Выполняется поиск объекта с максимальным годом.
12. Возвращается один объект.
